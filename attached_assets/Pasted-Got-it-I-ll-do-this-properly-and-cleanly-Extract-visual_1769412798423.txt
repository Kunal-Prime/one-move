# 📄 PRD — OneMove (Inspired UI Edition)

## Product Name

**OneMove**

---

## 1. Product Objective

OneMove helps users reduce mental overload by transforming unstructured thoughts into **one clear, immediate action**.

The product prioritizes **clarity, calmness, and decisiveness**.

---

## 2. Design Philosophy

* Dark, immersive interface
* Premium AI-product feel
* Calm over stimulation
* Minimal choices
* One dominant action per screen

---

## 3. Visual Design Requirements

### Theme

* Dark gradient background (deep charcoal → muted purple/blue)
* No flat black
* Soft depth effect

### Color Palette

* Primary accent: Soft neon blue or purple
* Text: Off-white / light gray
* Secondary elements: Muted grays
* Highlight color used sparingly

---

## 4. Typography

* Modern sans-serif font family
* High readability on dark backgrounds
* Clear hierarchy:

  * Large headline
  * Supporting subtext
  * Small helper text

No decorative or playful fonts allowed.

---

## 5. Layout Structure

### Main Screen

* Center-aligned content
* Large input area as focal point
* One primary CTA button
* No sidebar
* No multi-column layout

---

## 6. Interaction Flow

1. User lands on page
2. Sees calm, dark interface with single input focus
3. User enters thoughts
4. Clicks primary CTA
5. Subtle loading animation
6. Results appear as structured cards
7. Interaction ends

---

## 7. Output Presentation

Results are shown in **three clean cards**:

1. Core issue summary
2. Control vs non-control clarity
3. Single actionable step (highlighted)

Cards use:

* Slight elevation
* Soft borders
* Clear spacing
* Optional emojis for reinforcement only

---

## 8. Buttons & Actions

### Primary CTA

* Pill-shaped
* Solid accent color
* Soft glow or shadow
* Subtle hover animation

### Secondary Actions

* Outline style
* Lower visual priority

---

## 9. Motion & Animation

* Fade + slide transitions
* Smooth hover states
* No playful animations
* Motion must feel intentional

---

## 10. Accessibility

* High contrast text
* Readable font sizes
* Keyboard-friendly interactions

---

## 11. Non-Goals

* No dashboards
* No multiple modes
* No persistent chat
* No data history
* No user profiling

---

## 12. Success Criteria

* User understands value in < 10 seconds
* User feels calm enough to type honestly
* User leaves with one clear action
* Judges perceive polish and restraint

---

This UI style is **excellent** for OneMove **if** you:

* Don’t turn it into a “cool AI landing page”

You’re borrowing **visual confidence**, not complexity.
